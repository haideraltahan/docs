Mill is a unified evaluation pipeline that lets you benchmark AI models across every modality — text, image, video, and audio — through a single, consistent interface. Instead of juggling a different framework for each modality, Mill gives you one reproducible workflow that produces comparable results regardless of what you are evaluating. Whether you are stress-testing a language model on reasoning benchmarks, measuring an image classifier against standard vision suites, or assessing audio transcription quality, Mill handles the orchestration so you can focus on the results.

<Tip>
  Mill is an early-stage project under active development. APIs and CLI commands may change between releases. Pin your version in production and check the changelog before upgrading.
</Tip>

## Why Mill?

Modern AI systems rarely operate on a single modality. A production pipeline might rely on a language model for reasoning, a vision model for perception, and an audio model for transcription — and evaluating each one in isolation, with a different tool and a different reporting format, makes it nearly impossible to compare results or reproduce experiments.

Mill solves this by providing a single evaluation layer that speaks to all of them.

### One Pipeline for Every Modality

Mill ships with built-in support for **text**, **image**, **video**, and **audio** evaluation tasks. You configure your benchmark and your model in the same way regardless of modality. The same `mill evaluate` command — or `Evaluator.evaluate()` in Python — drives every run.

### Reproducible by Design

Every Mill evaluation captures the full configuration at run time: model identifiers, benchmark versions, preprocessing parameters, random seeds, and environment metadata. You can replay any past evaluation exactly, share a run configuration with a colleague, or diff two runs to understand what changed.

### Extensible Benchmarks and Models

Mill's benchmark and model registries are open for extension. Register a custom benchmark using the benchmark registration API, plug in your own model via the model registration API, or define new scoring logic through the custom metric interface. Point Mill at your model's inference endpoint or local weights and your custom setup participates in the same pipeline as every built-in component. No forking required.

<Note>
  Built-in benchmark coverage grows with each release. Check `mill list-benchmarks` for the current set available in your installed version.
</Note>
